HTML Introduction

HTML (HyperText Markup Language) is the foundation of web development. It provides the structure and semantic meaning to web content.

What is HTML?

HTML uses tags to define elements like headings, paragraphs, links, images, and more. It's a markup language, not a programming language.

HTML Syntax

Basic Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Page Title</title>
</head>
<body>
    <h1>Hello World</h1>
    <p>This is a paragraph.</p>
</body>
</html>

Core Concepts

Elements and Tags

HTML elements are defined by tags, which are keywords surrounded by angle brackets.

Attributes

Attributes provide additional information about HTML elements.

Semantic HTML

Using meaningful tags like <header>, <nav>, <main>, <article> improves accessibility and SEO.

Meta tags are invisible elements that provide information about the HTML document.

These tags include:

  • Character encoding: UTF-8
  • Viewport settings for responsive design
  • SEO description and keywords
  • Open Graph tags for social media sharing
  • Twitter Card meta tags
  • Favicon links
  • Canonical URL
  • External resource links (CSS, JS)

Note: Meta tags are processed by browsers and search engines, not displayed as visible content.

12. Data Attributes

<div data-user-id="123" data-role="admin" data-status="active" 
     style="border: 1px solid #ccc; padding: 10px; margin: 5px 0;"
     onclick="showDataAttributes(this)">
    User Information (Click to see data attributes)
</div>

<button data-action="delete" data-item-id="456" 
        onclick="handleAction(this)" style="margin: 5px;">
    Delete Item
</button>

<ul>
    <li data-category="electronics" data-price="299.99" 
        onclick="showProductInfo(this)" style="cursor: pointer; padding: 5px;">
        Laptop - Click for details
    </li>
    <li data-category="books" data-price="19.99" 
        onclick="showProductInfo(this)" style="cursor: pointer; padding: 5px;">
        Programming Book - Click for details
    </li>
</ul>

<script>
function showDataAttributes(element) {
    const userId = element.dataset.userId;
    const role = element.dataset.role;
    const status = element.dataset.status;
    alert(`User ID: ${userId}, Role: ${role}, Status: ${status}`);
}

function handleAction(button) {
    const action = button.dataset.action;
    const itemId = button.dataset.itemId;
    alert(`Action: ${action}, Item ID: ${itemId}`);
}

function showProductInfo(element) {
    const category = element.dataset.category;
    const price = element.dataset.price;
    alert(`Category: ${category}, Price: $${price}`);
}
</script>

Common Mistakes

1. Unclosed Tags and Improper Nesting

One of the most common HTML mistakes is forgetting to close tags or improperly nesting elements. Every opening tag must have a corresponding closing tag, and elements should be properly nested within each other. For example, always close tags in reverse order: <div><p>content</p></div>, not <div><p>content</div></p>. This ensures proper HTML structure and prevents rendering issues.

<!-- WRONG: Unclosed tags and improper nesting -->
<div><p>This paragraph is never closed
<div><span>Nesting error</div></span>

<!-- CORRECT: Properly closed and nested -->
<div>
    <p>This paragraph is properly closed</p>
    <span>Nesting is correct</span>
</div>

2. Missing DOCTYPE Declaration

The DOCTYPE declaration is crucial for telling browsers which version of HTML to use. Without <!DOCTYPE html>, browsers enter "quirks mode," which can cause inconsistent rendering across different browsers. Always include <!DOCTYPE html> as the very first line of your HTML document to ensure standards mode and consistent behavior.

<!-- WRONG: Missing DOCTYPE -->
<html>
    <head>...</head>
    <body>...</body>
</html>

<!-- CORRECT: Proper DOCTYPE -->
<!DOCTYPE html>
<html lang="en">
    <head>...</head>
    <body>...</body>
</html>

3. Using Deprecated Tags and Attributes

Many older HTML tags and attributes have been deprecated in HTML5 in favor of CSS. Avoid using tags like <font>, <center>, <marquee>, and attributes like bgcolor, border, and align. Instead, use modern semantic HTML elements and CSS for styling and layout. This improves accessibility, maintainability, and follows web standards.

<!-- WRONG: Deprecated tags and attributes -->
<font face="Arial" color="red" size="5">Red Text</font>
<center>Centered content</center>
<table border="1" cellpadding="5">

<!-- CORRECT: Modern HTML and CSS -->
<span style="font-family: Arial; color: red; font-size: 1.5em;">Red Text</span>
<div style="text-align: center;">Centered content</div>
<table style="border: 1px solid black; padding: 5px;">

4. Missing ALT Text for Images

Images without alt text create accessibility barriers for screen reader users and fail WCAG guidelines. The alt attribute provides alternative text for images when they cannot be displayed. Always include descriptive alt text for meaningful images and use empty alt="" for decorative images.

<!-- WRONG: Missing alt text -->
<img src="logo.jpg">
<img src="chart.png" alt=""> <!-- Missing description -->

<!-- CORRECT: Proper alt text -->
<img src="logo.jpg" alt="Company Logo">
<img src="chart.png" alt="Sales increased by 25% in Q3 2024">
<img src="decoration.png" alt=""> <!-- Decorative image -->

5. Improper Use of Heading Tags

Headings should be used hierarchically (h1, h2, h3, etc.) and not for styling text. Don't skip heading levels (jumping from h1 to h3 without h2), and don't use headings just to make text bold or larger. Proper heading structure is crucial for SEO, accessibility, and document outline.

<!-- WRONG: Improper heading structure -->
<h1>Main Title</h1>
<h3>Subsection</h3> <!-- Skipped h2 -->
<h5>Small text</h5> <!-- Used for styling -->

<!-- CORRECT: Hierarchical heading structure -->
<h1>Main Title</h1>
<h2>Main Section</h2>
<h3>Subsection</h3>
<p>Regular text for styling</p>

6. Forgetting Form Labels

Every form input must have an associated label for accessibility. Labels help screen readers identify form fields and improve usability by providing larger click targets. Use the for attribute to link labels to their corresponding input elements.

<!-- WRONG: Missing labels -->
Name: <input type="text" name="name">
<input type="checkbox" name="agree"> I agree

<!-- CORRECT: Proper labels -->
<label for="name">Name:</label>
<input type="text" id="name" name="name">

<input type="checkbox" id="agree" name="agree">
<label for="agree">I agree</label>
